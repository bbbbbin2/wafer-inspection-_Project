# ⚗️ 반도체 웨이퍼 맵 결함 자동 분류 시스템

> **반도체 웨이퍼 제조 공정의 미세 결함 검출을 위한 딥러닝 모델 구조 및 전처리 성능 비교 연구**

반도체 미세 공정이 고도화됨에 따라 사람의 육안이나 규칙 기반 알고리즘으로 식별하기 어려운 미세 결함을 자동으로 탐지하고 분류하기 위해 개발된 프로젝트입니다. 대용량 웨이퍼 맵 데이터를 활용하여 최적의 전처리 파이프라인을 구축하고, 3가지 딥러닝 아키텍처(CNN, ResNet18, EfficientNet-B0)의 구조적 특성에 따른 성능을 비교 분석했습니다.

---

## 1. 프로젝트 문제 정의 및 목표 (Problem Statement & Objectives)

### 문제 정의

* **기존 방식의 한계:** 반도체 웨이퍼 제조 공정에서 발생하는 미세 결함(스크래치, 오염, 패턴 이상 등)은 수율 감소 및 제품 신뢰도 저하의 주원인입니다. 기존의 육안 검사나 규칙 기반(Rule-based) 알고리즘은 **일관성이 부족하고 비용이 높다는 명확한 한계**가 있습니다.
* **핵심 해결 과제:** 1. 웨이퍼 이미지 데이터에서 결함 패턴을 자동으로 인식할 수 있는 최적의 데이터 전처리 방법 설계
2. CNN, ResNet18, EfficientNet-B0 등 다양한 구조의 모델 간 성능 비교
3. 동일한 실험 환경에서 모델별 성능 차이를 분석하여 구조적 특성과 성능의 관계 도출

### 프로젝트 목표

반도체 웨이퍼(Wafer) 이미지 데이터에서 발생하는 결함(Failure Type)을 자동으로 분류하는 딥러닝 모델을 개발하고, **동일 조건 하에 모델 구조 및 전처리 방식에 따른 성능을 정량적으로 비교 분석**하는 것을 목표로 합니다. (성능 평가 기준: $\text{F1-Score} > \text{Accuracy}$)

---

## 2. 데이터셋 정보 및 구조 (Dataset)

### 데이터셋 설명

* **출처:** Kaggle WM-811K Wafer Map Dataset
* **원본 데이터 규모:** 총 811,457장
* **데이터 형태 (Input):** WaferMap 2D 배열 형태
* `0`: 비활성 영역 (Wafer 외부 Empty Space)
* `1`: 정상 영역
* `2`: 결함 또는 패턴 정보 영역


* **출력 라벨 (Output):** 결함 유형 클래스 (총 9개 클래스 / *라벨링 데이터 기준*)
* *대표 결함 유형:* Donut, Edge-Ring, Loc, Center, Random, Scratch 등 및 None(정상)



### 데이터 분할 (Train / Validation / Test)

라벨이 누락된 데이터(`unknown`)를 제거한 **총 172,952장**의 유효 데이터를 기반으로 다음과 같이 분할하여 실험을 진행했습니다.

| 데이터셋 구분 | 비율 (%) | 데이터 수 (장) | 목적 |
| --- | --- | --- | --- |
| **Train Set** | 70% | 약 121,066 | 모델 가중치 학습 |
| **Validation Set** | 15% | 약 25,943 | 하이퍼파라미터 튜닝 및 과적합 방지 |
| **Test Set** | 15% | 약 25,943 | 최종 모델 성능 평가 및 일반화 성능 검증 |

---

## 3. 데이터 전처리 파이프라인 실험 (Pre-processing)

미세 노이즈 제거 및 불균형 데이터셋 학습 최적화를 위해 5개 단계(`P1`~`P5`)의 전처리 실험을 수행했습니다.

* **P1 (데이터 정제):** 811,457개의 2차원 데이터를 평탄화(Flatten)한 후 라벨 누락 데이터(`unknown`)를 제거하여 172,952개로 정제
* **P2 (Resize + 정규화):** 대용량 데이터 연산 효율성을 위해 $64 \times 64$ 크기로 규격화하고, $0 \sim 1$ 사이 값으로 Min-Max 정규화를 진행하여 가중치 폭주 방지 (**★ 최종 Baseline 채택**)
* **P3 (중앙값 필터 - Median Filter):** 공정 중 발생하는 무작위 '소금-후추 잡음(Salt-and-Pepper Noise)'을 제거하고 실제 불량 패턴의 형태를 보존
* **P4 (데이터 증강 - Augmentation):** 소수 불량 데이터를 Flip & Rotation(뒤집기 및 회전)하여 방향/위치에 무관한 결함 탐지력 확보
* **P5 (클래스 가중치 - Class Weight):** 정상(`None`) 데이터 편향을 막기 위해 소수 불량 데이터에 높은 벌점(가중치)을 부여하여 검출력 극대화

### 전처리 방법별 성능 비교 (동일 Baseline 모델 기준)

> 성능 평가 최우선 기준: **F1-Score**

| 전처리 방법 | Accuracy | F1-Score | Training Time (50 Epochs) |
| --- | --- | --- | --- |
| **P2 (Resize + 정규화) ★** | **97.16%** | **0.8478** | **5.37 분** |
| **P3 (중앙값 필터)** | 97.17% | 0.8365 | 5.37 분 |
| **P4 (데이터 증강)** | 97.23% | 0.8363 | 5.37 분 |
| **P5 (클래스 가중치)** | 95.86% | 0.8286 | 5.37 분 |

* **결론:** 분석 결과, **P2 (Resize + 정규화)** 가 가장 우수한 F1-Score를 기록하여 최종 모델 비교 분석용 전처리 데이터로 채택되었습니다.

## 4. 사용 모델 및 학습 조건 (Models & Hyperparameters)

### 모델 설명 및 구조적 특성

본 프로젝트는 **처음부터 학습(Scratch Train)** 방식을 채택하여 Pre-trained Weight를 사용하지 않고 동일한 데이터셋 환경에서 각 아키텍처 고유의 구조적 특성이 수율 분류에 미치는 영향을 직접 비교했습니다.

```
1. Custom CNN           [Conv Layer] ➔ [Pooling] ➔ [FC Layer] (기본적인 특징 추출)
2. ResNet18             [Input] ➔ [Residual Block (Skip Connection)] ➔ [Output] (지름길을 통한 깊은 신경망 안정성)
3. EfficientNet-B0      [MBConv] ➔ [Compound Scaling (Width/Depth/Resolution)] (최적의 리소스 효율성)

```

1. **Custom CNN:** 기본적인 합성곱(Convolution)과 풀링(Pooling) 레이어로 구성되어 웨이퍼의 공간적 특징을 포착하는 베이스라인 모델입니다.
2. **ResNet18:** **지름길 연결(Skip Connection)** 구조를 적용하여, 레이어가 깊어짐에 따라 발생하는 경사소실(Vanishing Gradient) 문제를 해결하고 안정적인 대용량 학습이 가능합니다.
3. **EfficientNet-B0:** 컴파운드 스케일링(Compound Scaling)을 통해 모델의 깊이, 너비, 입력 해상도를 균형 있게 조절하여 적은 파라미터로도 고효율·고성능을 내는 최적화 아키텍처입니다.

### 학습 하이퍼파라미터 (Hyperparameters)

모든 모델은 제어 변수를 통제하기 위해 **완전히 동일한 파라미터 환경**에서 학습되었습니다.

* **Input Size:** $64 \times 64$ (소수점 정규화 완료)
* **Classes:** 9개 (결함 유형 및 정상)
* **Batch Size:** 32
* **Epochs:** 50
* **Learning Rate:** 0.001
* **Optimizer:** AdamW (가중치 감쇠 적용으로 과적합 방지)
* **Loss Function:** CrossEntropyLoss

## 5. 최종 모델별 성능 비교 평가 (Evaluation)

최종 채택된 **P2 전처리 데이터**셋을 기반으로 50 Epoch 동안 학습을 진행한 결과입니다.

| Model | Accuracy | Precision | Recall | F1-score (★) | Inference Time |
| --- | --- | --- | --- | --- | --- |
| **Custom CNN** | 97.19% | 0.8776 | 0.8259 | 0.8498 | **20.5 min** |
| **ResNet18** | 97.28% | 0.8880 | **0.8607** | **0.8720** | 39.0 min |
| **EfficientNet-B0** | **97.47%** | **0.8916** | 0.8524 | 0.8702 | 69.5 min |

### 성능 결과 분석 및 도출

* **최적의 모델:** 본 프로젝트의 최우선 지표인 **F1-Score 기준**, ResNet18(0.8720)이 가장 우수한 성능을 보여 최종 서빙 모델로 적합하다고 판단됩니다.
* **구조적 특성과 성능의 관계:**
* **EfficientNet-B0**는 가장 높은 Accuracy(97.47%)와 Precision(0.8916)을 기록했으나, 복잡한 컴파운드 스케일링 구조로 인해 **Inference Time이 69.5분으로 가장 길어** 실시간 제어가 중요한 반도체 공정 라인 도입 시 비용적 트레이드오프(Trade-off)가 존재합니다.
* **ResNet18**은 Skip Connection 덕분에 소수 결함 클래스에 대한 **Recall(0.8607)이 가장 우수**했으며, 추론 시간(39분) 또한 EfficientNet 대비 약 1.8배 빨라 현업 적용 관점에서 가장 균형 잡힌 구조적 효율성을 증명했습니다.
